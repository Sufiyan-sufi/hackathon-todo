## Prompt

# Feature Specification: Mark as Complete

## Overview
Implement a function to toggle the completion status of an existing task in the in-memory Todo list by ID. This is the fifth basic level feature for Phase I, enabling users to mark tasks as complete or incomplete.

## Data Model
Reference: specs/data_model.md (shared across features)
- Class: Task
  - id: int (identifier for marking)
  - title: str
  - description: str
  - completed: bool (toggle this field)

The in-memory storage is a list[Task] in the main app module, shared with other features.

## Requirements
- Command: 'complete <id>' or 'mark <id>' (CLI input parsing, id as integer; toggle current status)
- Validate ID exists in list; if not, handle gracefully
- Toggle completed bool (True if incomplete, False if complete)
- Output confirmation: "Task marked as {complete/incomplete}: ID {id} - {title}"
- Integrate with existing list (find by ID, modify in place)

## Quality Gates (per Constitution Principle 5)
- Type-safe with full hints (e.g., def mark_complete(tasks: list[Task], task_id: int) -> None)
- Error handling: Raise ValueError if ID not found, print user-friendly message (e.g., "Task ID not found")
- Clean code: PEP 8, simple toggle logic, docstring for function
- Testable: Function modifies list in place, toggles correctly on multiple calls

- Invoke skill validate_spec to check completeness before code generation

## Implementation Notes for Claude Code
- Use Python 3.13+, no external deps
- Assume tasks list is passed as param (for modularity)
- No additional params needed; just toggle existing completed field
