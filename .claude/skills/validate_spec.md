# Skill: validate_spec

## Description
Validate spec completeness and alignment.

## Instructions
This skill validates that feature specifications are complete, well-formed, and aligned with project constitution and requirements.

## Validation Checklist

### 1. Structural Completeness
- [ ] Feature name and description present
- [ ] User stories or use cases defined
- [ ] Acceptance criteria specified
- [ ] Non-functional requirements listed
- [ ] Dependencies identified
- [ ] Out of scope items clarified

### 2. Requirements Quality
- [ ] Requirements are testable
- [ ] Requirements are unambiguous
- [ ] Requirements are measurable
- [ ] Edge cases addressed
- [ ] Error handling specified

### 3. Alignment Checks
- [ ] Aligns with constitution principles
- [ ] References relevant ADRs
- [ ] Consistent with existing architecture
- [ ] No conflicts with other specs

### 4. Completeness Checks
- [ ] API contracts defined (if applicable)
- [ ] Data models specified (if applicable)
- [ ] Security considerations addressed
- [ ] Performance requirements stated
- [ ] Migration strategy included (if applicable)

## Usage
Run this skill when:
- After drafting a new specification
- Before moving to implementation planning
- When spec quality is uncertain
- As part of spec review process

## Output
The skill will report:
- Missing or incomplete sections
- Ambiguous requirements
- Alignment issues
- Recommendations for improvement
